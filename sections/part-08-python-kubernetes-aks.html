3Ô∏è‚É£ Authentication & kubeconfig

from kubernetes import config

config.load_kube_config()

This loads credentials from:

  • ~/.kube/config
  • AKS credentials retrieved via Azure CLI

4Ô∏è‚É£ Listing Pods with Python

from kubernetes import client, config

config.load_kube_config()
v1 = client.CoreV1Api()

pods = v1.list_namespaced_pod(namespace="default")
for pod in pods.items:
    print(pod.metadata.name)

This script does exactly what kubectl get pods does, but programmatically.

5Ô∏è‚É£ Creating Kubernetes Resources

deployment = {
  "apiVersion": "apps/v1",
  "kind": "Deployment",
  "metadata": {"name": "python-app"},
  "spec": {
    "replicas": 2,
    "selector": {"matchLabels": {"app": "python"}},
    "template": {
      "metadata": {"labels": {"app": "python"}},
      "spec": {
        "containers": [
          {
            "name": "app",
            "image": "nginx"
          }
        ]
      }
    }
  }
}

This dictionary can be submitted directly to the Kubernetes API.

6Ô∏è‚É£ Watching Cluster Events

from kubernetes import watch

w = watch.Watch()
for event in w.stream(v1.list_namespaced_pod, namespace="default"):
    print(event["type"], event["object"].metadata.name)

Watching resources is useful for:

  • Monitoring deployments
  • Triggering alerts
  • Building operators

7Ô∏è‚É£ Python with AKS (Azure Integration)

Python also integrates with Azure using the Azure SDK. 

pip install azure-identity azure-mgmt-containerservice
  
from azure.identity import DefaultAzureCredential
from azure.mgmt.containerservice import ContainerServiceClient

credential = DefaultAzureCredential()
client = ContainerServiceClient(credential, "SUBSCRIPTION_ID")

This allows you to:

  • Create AKS clusters
  • Scale node pools
  • Query cluster status

8Ô∏è‚É£ Scaling AKS Node Pools with Python

client.agent_pools.begin_create_or_update(
  resource_group_name="rg-aks",
  resource_name="aks-cluster",
  agent_pool_name="nodepool1",
  parameters={
    "count": 5,
    "vm_size": "Standard_DS2_v2"
  }
)

This is equivalent to scaling via Azure Portal or CLI, but fully automated.
